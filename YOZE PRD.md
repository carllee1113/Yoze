## 1. 产品概述 (Product Overview)

### 产品名称
**YOZE 藥師**（暂定）

### 一句话描述
一款专为老年患者设计的服药提醒 App，以彩虹颜色直观显示服药进度，配合家庭看护者实时提醒功能，确保认知能力下降的老人不错过任何一次用药。

### 核心价值主张
> "让服药像看彩虹一样简单 — 红色吃、蓝、绿安心。"

### MVP 范围
- **App Only**（不含硬件）
- **平台：** Flutter（iOS + Android）
- **语言：** 简体中文（界面）+ 中文语音播报

---

## 2. 设计目标 (Design Goals)

### 2.1 可访问性（A11y-first）
- 字体 ≥ 18pt（正文），标题 ≥ 24pt
- 按钮高度 ≥ 56dp（老人手指不精准）
- 所有颜色按钮/图标须有文字标签辅助
- 高对比度（WCAG AA 标准）
- **语音播报**：每次提醒响起时，同步语音朗读"该吃药了"

### 2.2 简洁性
- 单屏不超过 3 个主要操作
- 最多 2 次点击完成任何核心任务
- 无专业术语，全用口语（如"吃药"而非"服药"）

### 2.3 家庭联动
- 看护者（子女）可通过 App 实时掌握老人服药状态
- 漏服时自动推送给看护者，无需老人手动操作

## 3. 目标用户 (Target Users)

### 3.1 主要用户画像

| | 患者端 | 看护者端 |
|---|---|---|
| **典型用户** | 父亲，72岁 | Carl，45岁，在深圳工作 |
| **健康状况** | 认知能力下降 + 糖尿病 + 高血压 | 健康 |
| **设备能力** | 会用微信，看不懂英文 app | 熟悉智能手机 |
| **主要痛点** | 忘记吃药 / 记不清吃没吃 | 不知道爸爸有没有吃药，远程担心 |
| **使用场景** | 每天 4 次（早/午/晚/睡前）收到提醒 | 工作间隙查看通知 + 确认状态 |

### 3.2 用户场景

**场景 1：正常服药日**
```
08:00 → 红色提醒响起 → 父亲点"已吃" → 变黄色 → 完成
12:00 → 橙色提醒响起 → 父亲点"已吃" → 变蓝色 → 完成
18:00 → 黄色提醒响起 → 父亲点"已吃" → 变绿色 → 完成
22:00 → 青色提醒响起 → 父亲点"已吃" → 变紫色 → 完成
```

**场景 2：漏服**
```
08:00 → 红色提醒响起 → 父亲没确认
08:30 → 系统检测漏服 → Carl 手机收到推送："爸爸还没吃早上那次药"
```

**场景 3：看护者添加药物**
```
Carl 打开 App → 点"+" → 输入：
  药名：二甲双胍
  时间：08:00 / 20:00
  颜色标签：红色（早）/ 蓝色（晚）
→ 确认 → 同步到父亲手机上
```

| 功能编号 | 功能名称 | 优先级 | 说明 |
|---------|---------|--------|------|
| F01 | 添加药物 | P0 | 药名 + 时间 + 剂量 |
| F02 | 彩虹服药提醒 | P0 | 按 RAINBOW 系统显示颜色进度 |
| F03 | 确认吃药 | P0 | 点一下确认，方便老人操作 |
| F04 | 看护者绑定 | P0 | 子女账号绑定老人，实现双向通知 |
| F05 | 漏服提醒 | P0 | 超出时间窗口未确认 → 推送看护者 |
| F06 | 服药历史 | P1 | 日历视图，看过去 7/30 天的服药记录 |
| F07 | 语音播报 | P1 | 提醒响起时同步语音朗读 |
| F08 | 闹钟声音 | P0 | 即使手机静音，也响铃提醒 |

### 4.2 RAINBOW 服药颜色系统（核心亮点）🌈

#### 设计原理
老人记不住"第几次吃药"，但能记住颜色。用 7 色彩虹代表服药进度，**颜色即状态，颜色即信心**。

#### 每日剂量与颜色对应

```
假设每天最多吃 4 次药：

⏰ 08:00 第一次 → 🔴 红色（Red）
⏰ 12:00 第二次 → 🟡 黄色（Yellow）
⏰ 18:00 第三次 → 🔵 蓝色（Blue）
⏰ 22:00 第四次 → 🟢 绿色（Green）

（若每天 >4 次，第 5 次循环回红色）
```

#### 颜色状态流转

```
待提醒（未到时间）→ 灰色
提醒中（到点未确认）→ 亮红色（闪烁动画）
已确认 → 变色（按序列：黄→蓝→绿→紫→橙→青→红）
漏服 → 红色 + 感叹号 ⚠️
```

#### 视觉设计要求
- 每种颜色的色值需足够鲜明，确保老人一眼能区分
- 确认后颜色"渐变"动画（0.5s）而非突然切换，增加正向反馈感
- 当前需要吃药时：颜色图标放大 + 轻微脉冲动画

#### 颜色色值建议（高对比）

| 颜色 | HEX | 用途 |
|------|-----|------|
| 🔴 红 | `#E53935` | 第1次 / 漏服警告 |
| 🟠 橙 | `#FB8C00` | 第5次 |
| 🟡 黄 | `#FDD835` | 第2次 / 已确认 |
| 🟢 绿 | `#43A047` | 第4次 / 已完成全天 |
| 🔵 蓝 | `#1E88E5` | 第3次 |
| 🟣 紫 | `#8E24AA` | 第6次 |
| 🩵 青 | `#00ACC1` | 第7次 |

### 4.3 提醒系统

#### 提醒触发逻辑
```
T = 服药时间

T - 0min   → 推送 + 响铃 + 语音
T + 30min  → 未确认 → 重复响铃
T + 60min  → 未确认 → 推送通知给看护者
T + 120min → 未确认 → 看护者收到第二次强提醒
```

#### 通知内容
- **患者端（老人）：** "🔴 该吃药了！"
- **看护者端：** "⚠️ 爸爸还没吃 [药名]（红色，第1次）"

### 4.4 服药历史

- 界面：日历流（似微信聊天记录形式）
- 每条记录显示：时间 / 药名 / 颜色标签 / 是否漏服
- 支持 7 天 / 30 天切换
- 看护者可看老人完整历史，患者端仅看自己记录

## 7. 技术需求概述 (Technical Requirements)

### 7.1 技术栈
- **框架：** Flutter 3.x
- **目标平台：** iOS 12+ / Android 8+
- **状态管理：** Riverpod / Provider
- **本地存储：** SQLite（服药历史）+ SharedPreferences（设置）
- **推送服务：** Firebase Cloud Messaging (FCM) + 厂商通道
- **实时同步：** Firebase Realtime Database 或类似

### 7.2 关键实现点
- **后台提醒：** 使用 flutter_local_notifications（支持后台定时）
- **闹钟穿透静音：** iOS 需使用 SoundType = default; Android 需申请 USE_FULL_SCREEN_INTENT 权限
- **彩虹颜色动画：** 使用 AnimatedContainer 或 custom painter
- **看护者同步：** Firebase Realtime DB（轻量，适合 MVP）

### 7.3 推送通知设计
```
{
  "type": "medication_reminder",
  "patient_id": "xxx",
  "medicine_name": "二甲双胍",
  "dose_time": "08:00",
  "color": "red",
  "sequence": 1
}

{
  "type": "missed_dose_alert", 
  "patient_id": "xxx",
  "caregiver_id": "yyy",
  "medicine_name": "二甲双胍",
  "missed_at": "08:30"
}
```

## 8. MVP 完成定义 (Definition of Done)
